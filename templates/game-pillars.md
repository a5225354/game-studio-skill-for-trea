## 什么是游戏支柱？

支柱是定义此游戏身份的 3-5 个不可协商的原则。
每个设计、美术、音频、叙事和技术决策都必须服务于至少一个支柱。
如果一个功能不服务于任何支柱，它就不属于这个游戏。

**为什么支柱重要**: 在典型的开发周期中，团队会做出成千上万个
小型创意决策。支柱确保所有这些决策朝同一个方向推进，
创造一致的玩家体验而非断开的功能集合。

### 什么是一个好支柱

好支柱的特征:
- **可证伪性**: "有趣的玩法"不是支柱。"战斗奖励耐心而非冲动"是 — 它对设计选择做出了可测试的主张。
- **约束性**: 如果一个支柱从不迫使你对某些东西说不，它就太模糊了。好支柱会排除选项。
- **跨部门性**: 一个只约束游戏设计但对美术、音频或叙事没有影响的支柱是不完整的。真正的支柱影响每个学科。
- **可记忆性**: 团队应该能凭记忆背诵支柱。如果做不到，说明支柱太多或太复杂。

### 真实的 AAA 示例

这些工作室公开分享了他们的游戏支柱，展示了有效支柱可以多么具体:

| 游戏 | 支柱 | 为什么有效 |
| ---- | ---- | ---- |
| **战神 (2018)** | 残暴的战斗；父子情感旅程；连续镜头 (无切换)；北欧神话重新演绎 | "连续镜头"是激进的 — 它削减了标准的电影工具。"父子旅程"约束了叙事、关卡设计，以及战斗 (阿特柔斯作为同伴)。 |
| **哈迪斯** | 快速流畅的战斗；通过重复展现叙事深度；每次运行都教给玩家新东西 | "通过重复展现故事"在叙事上为 Roguelike 结构提供了合理性 — 死亡就是故事。"每次运行都教给新东西"约束了关卡和遭遇设计。 |
| **最后的我们** | 故事是必不可少的，不是可选的；AI 伙伴建立关系；始终可以选择潜行 | "AI 伙伴建立关系"推动了同伴 AI 的大量投入 — 不仅是寻路，而是情感存在。 |
| **蔚蓝** | 艰难但公平；无妥协的无障碍设计；故事和机制是同一件事 | "故事和机制是同一件事" — 攀爬就是挣扎，冲刺就是焦虑。支柱防止了机制变成"仅仅是玩法"。 |
| **空洞骑士** | 氛围胜过解释；赢得的精通；世界讲述自己的故事 | "氛围胜过解释" — 没有教程，没有手把手引导，世界通过环境设计来教导。 |
| **死亡细胞** | 每把武器都可行；战斗是一场舞蹈；永久死亡创造意义 | "每把武器都可行"是极具约束性的 — 它要求在数百个物品上进行持续的平衡工作。 |

## 反支柱 (此游戏不是什么)

反支柱与支柱同样重要 — 它们防止范围蔓延并保持愿景聚焦。每一个"不"保护着"是"。

好的反支柱是团队可能真的想做的事情。"不是一个竞速游戏"是显而易见和无用的。
"不是一个开放世界游戏"在类型可能支持的情况下是有用的。

## 支柱冲突解决

当两个支柱冲突时 (它们会的)，使用此优先级顺序。排名反映了体验的哪些方面对核心幻想最为关键。

| 优先级 | 支柱 | 理由 |
| ---- | ---- | ---- |
| 1 | [Highest priority pillar] | [Why this wins when it conflicts with others] |
| 2 | [Second priority] | [Why] |
| 3 | [Third priority] | [Why] |

**解决流程**:
1. 确定哪些支柱存在张力
2. 查阅上方的优先级排名
3. 如果低优先级支柱可以在不损害高优先级支柱的情况下被部分满足，则这样做
4. 如果不行，高优先级支柱胜出
5. 在相关设计文档中记录决策和理由
6. 如果冲突是根本性的 (两个支柱不可调和)，上报给 creative-director 考虑修订支柱本身

## 支柱验证清单

在最终确定支柱之前，验证:

- [ ] **数量**: 3-5 个支柱 (不多不少)
- [ ] **可证伪性**: 每个支柱做出了可能错误的声明
- [ ] **约束性**: 每个支柱迫使对某些合理的想法说不
- [ ] **跨部门性**: 每个支柱对设计、美术、音频、叙事和工程都有影响
- [ ] **设计测试**: 每个支柱有一个解决真实决策的具体设计测试
- [ ] **反支柱已定义**: 至少 3 条明确的"此游戏不是"陈述
- [ ] **优先级排序**: 解决支柱间冲突的明确顺序
- [ ] **MDA 对齐**: 支柱共同传达排名靠前的目标美学
- [ ] **SDT 覆盖**: 至少一个支柱服务自主性，一个服务能力感，一个服务归属感
- [ ] **可记忆性**: 团队能凭记忆背诵所有支柱
- [ ] **核心幻想已服务**: 每个支柱都追溯到核心幻想承诺
