# 安全策略

## 支持的版本

仅 `main` 分支接受安全修复。Fork 和旧版本不受支持。

## 报告漏洞

**请勿通过公开的 GitHub Issue 报告安全漏洞。**

请使用 GitHub 的私有漏洞报告功能：

**[报告漏洞 →](https://github.com/Donchitos/Claude-Code-Game-Studios/security/advisories/new)**

请尽可能提供以下详细信息：
- 漏洞描述及其影响范围
- 复现步骤
- 潜在影响和攻击场景
- 任何建议的缓解措施

**预期响应时间：**
- **48 小时内**确认收到
- **7 天内**提供状态更新
- 确认漏洞后 **90 天内**完成修复

## 哪些在范围内

本 Skill 是**本地开发工具** — 它提供 PowerShell 脚本和 AI 协作指令，直接在你的
本地计算机上运行。安全问题主要涉及可能在你不知情的情况下在你的环境中执行的代码。

### 高严重性
- 钩子脚本（`hooks/*.ps1`）中执行恶意或未公开的 PowerShell 命令
- 技能或代理指令中窃取环境变量、API 密钥或机密信息的行为
- 通过技能或代理定义进行的提示词注入，导致 AI 绕过安全措施或执行未经授权的破坏性操作
- 悄然改变行为、用户无法审计的贡献

### 中等严重性
- 技能中包含未公开的对外网络请求
- 代理定义中升级权限或绕过用户确认提示的行为
- 钩子模式在不同平台上表现不一致以隐藏行为
- 技能在未获得明确用户批准步骤的情况下写入超出其文档范围的内容

### 不在范围内
- Trae IDE 或底层 AI 模型本身的漏洞（请报告给 Trae 平台提供商）
- 用户 Trae IDE 安装或编辑器扩展中的 Bug
- 没有实际攻击路径的理论漏洞
- 需要物理访问用户计算机的问题

## 贡献者安全指南

在贡献钩子、技能或代理时：

- **钩子必须使用 PowerShell** — 使用 `Write-Host` 而非 `echo`；避免平台特定语法
- **禁止静默网络请求** — 钩子或技能中的网络请求必须明确记录并需用户主动选择
- **禁止读取机密或环境变量** — 除非是技能能正常运行所必需且已在技能头部明确记录的最低限度
- **技能不得写入超出其文档范围的内容** — 除非有明确的用户确认步骤

## 披露政策

我们遵循 **90 天协调披露**时间表：

1. 你私下提交漏洞
2. 我们在 48 小时内确认收到
3. 我们在 7 天内确认并评估严重性
4. 我们开发并测试修复方案
5. 我们在公开披露前通知你
6. 公开披露在修复发布后进行，或最迟在 90 天时进行

我们会在发布说明中致谢报告者，除非你希望保持匿名。
