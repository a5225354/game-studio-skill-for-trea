# Soak Test — 浸泡测试

浸泡测试是在特定观察目标下运行的扩展游戏会话。它运行 **30分钟到数小时** 以暴露：内存泄漏、性能漂移、状态累积Bug、乐趣疲劳、内容耗尽。

**本技能生成观察协议和分析工具 — 人类执行实际玩测。**

**输出:** `production/qa/soak-test-[date]-[duration].md`

---

## 触发条件

- 用户说"浸泡测试" / "soak test" / "长时间测试" / "耐久测试"

---

## 1. 解析参数

**时长** (默认 `1h`): `30m` / `1h` / `2h` / `4h`

**焦点** (默认 `all`): `memory` / `stability` / `balance` / `all`

---

## 2. 加载上下文

Read: `docs/technical-preferences.md`(性能预算)、`design/gdd/game-concept.md`(设计会话时长)、最近试玩记录、当前QA计划。

---

## 3. 定义观察检查点

基于时长生成定时检查点：30m→T+0,10,20,30 / 1h→T+0,15,30,45,60 / 2h→7点 / 4h→7点。

---

## 4. 生成浸泡测试协议

按焦点生成观察项目：

**内存/稳定性** — 引擎特定监控指导（Godot: Debugger→Monitors；Unity: Memory Profiler；Unreal: stat memory）

**稳定性** — 无崩溃/卡死、帧率在预算内、音频正常、HUD正常、输入正常

**平衡/疲劳** — 核心机制仍奖励感、难度感知、内容耗尽迹象、挫折/高潮时刻

---

## 5. 生成协议文档

包含：会话前设置 → 基线(T+0) → 检查点日志(每个时间点) → 会话后分析(内存趋势、稳定性摘要、平衡/疲劳摘要、发现的问题) → Verdict: PASS/CONCERNS/FAIL

---

## 6. 写入输出

征求意见后写入 `production/qa/soak-test-[date]-[duration].md`。

---

## 协作协议

- **生成协议 — 人类执行** — 绝不尝试自动运行浸泡测试
- **时长应匹配游戏会话设计**
- **首次浸泡应为 `all` 焦点**
- **写入前询问**

Verdict: **COMPLETE** — 浸泡测试协议已写入。
