# Balance Check — 游戏平衡分析技能

触发条件:
- 用户说"平衡检查" / "balance" / "数值平衡" / "平衡性" / "balance check"
- 用户指定了系统名称或数据文件路径

---

当用户触发此技能时:

## 1. 识别平衡领域

从参数中提取系统名称或数据文件路径。

## 2. 读取相关数据文件

从 `assets/data/` 和 `design/balance/` 中获取相关数据文件。

## 3. 读取设计文档

从 `design/gdd/` 中获取被检查系统的设计文档。

## 4. 执行分析

### 战斗平衡
- 计算每个威力等级所有武器/技能的 DPS
- 检查每个等级的击杀时间（TTK）
- 识别是否存在严格优于其他选项的选项（支配性选项）
- 检查防御选项是否能创造无敌状态
- 验证伤害类型/抗性交互是否平衡

### 经济平衡
- 映射所有资源产出源和消耗点及其流速
- 预测资源随时间的累积情况
- 检查是否存在无限资源循环
- 验证金币消耗是否与金币产出同步增长

### 进度平衡
- 绘制经验值曲线和战力曲线
- 检查是否存在空白区（长时间无实质性进度）
- 检查是否存在战力飙升（能力突然大幅跃升）
- 验证内容门槛是否与预期玩家战力对齐

### 战利品平衡
- 计算获取每个稀有度等级的预期时间
- 检查保底计时器的数学模型
- 验证没有任何战利品在任意阶段完全无用

## 5. 输出分析报告

```
## 平衡性检查：[系统名称]

### 已分析的数据来源
- [已读取的文件列表]

### 健康摘要：[健康 / 存在隐患 / 严重问题]

### 检测到的异常值
| 项目/数值 | 预期范围 | 实际值 | 问题 |
|-----------|---------|--------|------|

### 发现的退化策略
- [策略描述及问题所在]

### 建议
| 优先级 | 问题 | 建议修复 | 影响 |
|--------|------|---------|------|

### 需要关注的数值
[具体数值及建议调整和理由]
```

## 协作协议

1. 读取数据和设计文档
2. 执行数学模型分析
3. 展示分析结果和建议
4. 此技能为分析模式 — 展示发现，不修改文件
