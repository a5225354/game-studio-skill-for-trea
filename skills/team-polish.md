# Team Polish — 打磨团队编排技能

触发条件:
- 用户说"打磨团队" / "team polish" / "polish" / "打磨" / "优化品质"
- 用户提供了需要打磨的功能或区域描述

---

当用户触发此技能时，通过结构化流水线编排打磨流程。采用**顺序阶段角色扮演**模式。

**决策点**: 在每个阶段转换时，使用 `AskUserQuestion` 将方案作为可选项呈现给用户。用户批准后方可进入下一阶段。

## 团队组成
- **performance-analyst（性能分析师）** — 性能分析、优化、内存分析、帧预算
- **technical-artist（技术美术）** — 视觉特效打磨、着色器优化、视觉品质
- **sound-designer（音效设计师）** — 音频打磨、混音、环境音层、反馈音效
- **qa-tester（QA 测试员）** — 边界情况测试、回归测试、长时间压力测试

## 角色激活方式

在每个阶段，读取对应的角色文件并扮演该角色：
- `agents/performance-analyst.md` → 性能分析与优化
- `agents/technical-artist.md` → 视觉打磨
- `agents/sound-designer.md` → 音频打磨
- `agents/qa-tester.md` → 加固测试

使用 `TodoWrite` 跟踪各阶段进度。

## 流水线

### 阶段 1：评估
激活 **performance-analyst** 角色：
- 对目标功能/区域进行性能分析
- 识别性能瓶颈和帧预算违规
- 测量内存使用情况并检查内存泄漏
- 对照目标硬件规格进行基准测试
- 使用 AskUserQuestion 确认优化优先级
- 输出：附带优先级排序的优化清单的性能报告

### 阶段 2：优化
激活 **performance-analyst** 角色：
- 修复阶段 1 中发现的性能热点
- 优化绘制调用，减少过度绘制
- 修复内存泄漏并降低分配压力
- 验证优化未改变游戏行为
- 输出：附带优化前后指标的优化代码

### 阶段 3：视觉打磨
激活 **technical-artist** 角色：
- 审查视觉特效的品质和与美术圣经的一致性
- 优化粒子系统和着色器效果
- 在适当位置添加屏幕震动、镜头效果和视觉冲击感
- 确保效果在较低设置下优雅降级
- 输出：打磨后的视觉效果

### 阶段 4：音频打磨
激活 **sound-designer** 角色：
- 审查音频事件的完整性（是否有任何操作缺少声音反馈？）
- 检查音频混音级别 — 没有任何内容相对于整体混音过大或过小
- 添加环境音层以营造氛围
- 验证音频在空间定位下正确播放
- 输出：音频打磨清单和混音说明

### 阶段 5：加固
激活 **qa-tester** 角色：
- 测试所有边界情况：边界条件、快速输入、异常操作序列
- 长时间压力测试：长时间运行该功能，检查是否存在性能退化
- 压力测试：最大实体数量、最坏情况场景
- 回归测试：验证打磨变更未破坏现有功能
- 在最低规格硬件上进行测试（如可用）
- 输出：测试结果及剩余问题

### 阶段 6：签核
- 收集所有阶段的结果
- 将性能指标与预算进行对比
- 报告：可以发布 / 需要更多工作
- 列出所有剩余问题及其严重程度和建议

## 输出
一份涵盖以下内容的总结报告：性能前后指标、视觉打磨变更、音频打磨变更、测试结果以及发布就绪评估。

## 协作协议
1. 每阶段开始前读取对应角色文件
2. 使用 AskUserQuestion 呈现关键决策选项
3. 使用 TodoWrite 跟踪进度
4. 写入任何文件前获得用户批准
