# Onboard — 入职文档生成技能

触发条件:
- 用户说"入门" / "onboard" / "新成员" / "新人" / "入职"

---

当用户触发此技能时:

## 1. 读取项目上下文

读取 SKILL.md 获取项目概述和标准。

## 2. 读取相关角色定义

如果指定了特定角色，读取 `agents/` 中对应的角色文件。

## 3. 扫描代码库的相关领域

- 程序员：扫描 `src/` 的架构、模式、关键文件
- 设计师：扫描 `design/` 的现有设计文档
- 叙事：扫描 `design/narrative/` 的世界观和故事文档
- QA：扫描 `tests/` 的现有测试覆盖率
- 制作人：扫描 `production/` 的当前冲刺和里程碑

## 4. 生成入职文档

```markdown
# 入职指南：[角色/领域]

## 项目概述
[2-3 句话概述这个游戏是什么以及当前状态]

## 你的角色
[这个角色在本项目中的职责和关键责任]

## 项目架构
[与该角色相关的架构概述]

### 关键目录
| 目录 | 内容 | 你的交互方式 |
|------|------|------------|

### 关键文件
| 文件 | 用途 | 阅读优先级 |
|------|------|-----------|

## 当前标准和规范
[与该角色相关的规范摘要]

## 你所在领域的当前状态
[已完成的工作、正在进行的工作、接下来计划的]

## 常见陷阱
[新贡献者容易踩坑的地方]

## 首批任务
1. [先阅读这些文档]
2. [审查此代码/内容]
3. [从这个小任务开始]
```

## 协作协议

1. 读取项目上下文和目标角色
2. 扫描相关代码和文档
3. 生成入职文档
4. 写入前获得用户批准
