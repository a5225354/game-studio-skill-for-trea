# Team Level — 关卡团队编排技能

触发条件:
- 用户说"关卡团队" / "team level" / "关卡设计" / "设计关卡" / "创建区域"
- 用户提供了关卡名称或区域描述，如 "用关卡团队设计新手教程"

---

当用户触发此技能时，通过结构化流水线编排关卡开发。采用**顺序阶段角色扮演**模式。

**决策点**: 在每个步骤转换时，使用 `AskUserQuestion` 将方案作为可选项呈现给用户。用户批准后方可进入下一步。

## 前期收集上下文

1. 读取 `design/gdd/game-concept.md` 获取游戏概念
2. 读取 `design/gdd/game-pillars.md` 获取游戏支柱
3. 读取 `design/levels/` 中现有关卡文档
4. 读取 `design/narrative/` 中相关叙事文档
5. 读取该区域所属地区/阵营的世界观设定文档

## 团队组成
- **narrative-director（叙事总监）** — 叙事目的、角色、情感弧线
- **world-builder（世界构建师）** — 背景设定上下文、环境叙事、世界规则
- **level-designer（关卡设计师）** — 空间布局、节奏、遭遇战、导航
- **systems-designer（系统设计师）** — 敌人组合、战利品表、难度平衡
- **art-director（美术总监）** — 视觉主题、调色板、光照、资产需求
- **qa-tester（QA 测试员）** — 测试用例、边界测试、试玩检查清单

## 角色激活方式

在每个步骤，读取对应的角色文件并扮演该角色：
- `agents/narrative-director.md` → 叙事上下文
- `agents/world-builder.md` → 世界背景
- `agents/level-designer.md` → 布局与遭遇战
- `agents/systems-designer.md` → 系统集成
- `agents/art-director.md` → 视觉方向
- `agents/qa-tester.md` → QA 规划

使用 `TodoWrite` 跟踪各步骤进度。

## 流水线（顺序执行）

### 步骤 1：叙事上下文
激活 **narrative-director** 角色：
- 定义该区域的叙事目的（这里发生哪些故事节拍？）
- 识别关键角色、对话触发点和背景设定元素
- 指定情感弧线（玩家进入时、探索中、离开时应有什么感受？）

激活 **world-builder** 角色：
- 提供该区域的背景设定上下文（历史、阵营势力、生态）
- 定义环境叙事的机会
- 指定影响该区域玩法的世界规则

### 步骤 2：布局与遭遇战设计
激活 **level-designer** 角色：
- 设计空间布局（关键路径、可选路径、隐藏区域）
- 定义节奏曲线（紧张高峰、休息区域、探索区域）
- 放置具有难度递进的遭遇战
- 设计环境谜题或导航挑战
- 定义兴趣点和地标以辅助寻路
- 指定入口/出口点及与相邻区域的连接

### 步骤 3：系统集成
激活 **systems-designer** 角色：
- 指定敌人组合和遭遇战公式
- 定义战利品表和奖励放置
- 根据玩家预期等级/装备平衡难度
- 设计区域专属机制或环境危害
- 指定资源分布（生命恢复道具、存档点、商店）

### 步骤 4：视觉方向
激活 **art-director** 角色：
- 定义该区域的视觉主题和调色板
- 指定光照氛围和时间设置
- 列出所需美术资产（环境道具、专属资产）
- 定义视觉地标和视线
- 指定特殊视觉特效需求（天气、粒子、雾气）

### 步骤 5：QA 规划
激活 **qa-tester** 角色：
- 编写关键路径的测试用例
- 识别边界情况和极端情况（顺序越界、软锁定）
- 创建该区域的试玩检查清单
- 定义关卡完成的验收标准

## 输出

1. 编制关卡设计文档，将所有步骤输出整合为关卡设计模板格式
2. 保存至 `design/levels/[关卡名称].md`
3. 输出摘要，包含：区域概述、遭遇战数量、预估资产列表、叙事节拍，以及跨团队依赖关系或待解决问题

## 协作协议
1. 每步骤开始前读取对应角色文件
2. 使用 AskUserQuestion 呈现关键决策选项
3. 使用 TodoWrite 跟踪进度
4. 写入任何文件前获得用户批准
