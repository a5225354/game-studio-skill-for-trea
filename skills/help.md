# Studio Help — 下一步做什么

本技能为只读技能 — 仅报告发现，不写入任何文件。

本技能精确判断你在游戏开发管线中的位置，并告诉你下一步该做什么。它是**轻量级**的 — 不是完整审计。如需完整差距分析，请使用 project-stage-detect 技能。

---

## 触发条件

- 用户说"我该做什么" / "下一步做什么" / "卡住了" / "不知道做什么"
- 用户说"完成了 XXX，接下来呢？"

---

## 步骤1: 读取目录

使用 Read 工具读取 `docs/workflow-catalog.yaml`。这是所有阶段、步骤及其完成产物的权威列表。

---

## 步骤1b: 查找未编目技能

在读取目录后，Glob `skills/*.md` 获取已安装的完整技能列表。对比目录中的已编目命令，找出未编目的技能并在输出中展示。

---

## 步骤2: 确定当前阶段

按以下顺序检查：

1. **Read `production/stage.txt`** — 如果存在且有内容，就是权威阶段名
2. **如果 stage.txt 缺失**，从产物推断阶段（取最高匹配）：
   - `src/` 有10+源文件 → Production
   - `production/stories/*.md` 存在 → Pre-Production
   - `docs/architecture/adr-*.md` 存在 → Technical Setup
   - `design/gdd/systems-index.md` 存在 → Systems Design
   - `design/gdd/game-concept.md` 存在 → Concept
   - 什么都没有 → Concept（新项目）

---

## 步骤3: 读取会话上下文

Read `production/session-state/active.md`（如果存在），提取最近工作内容和进行中任务。

---

## 步骤4: 检查当前阶段各步骤完成情况

根据步骤2确定的阶段和步骤1的目录定义，逐步骤检查产物是否存在：
- 如果有 `artifact.glob` — Glob 检查匹配文件是否存在
- 标记 Complete / Incomplete / MANUAL（无法自动检测）

**特殊情况 — Production阶段**：优先读取 `production/sprint-status.yaml` 获取精确故事状态。

---

## 步骤5: 定位并识别下一步

1. **最后确认完成的步骤** — 最远的已完成必需步骤
2. **当前阻塞项** — 第一个未完成的必需步骤
3. **可选机会** — 可以同时进行的可选步骤

---

## 步骤6: 检查进行中的工作

如果 `active.md` 显示活跃任务，在输出顶部突出显示。

---

## 步骤7: 呈现输出

保持**简短直接**。使用以下格式：

```
## 你的位置: [阶段名]

**进行中:** [来自active.md，如有]

### ✓ 已完成
- [完成步骤]
- [完成步骤]

### → 下一步 (必需)
**[步骤名]** — [描述]
执行方式: [技能名]

### ~ 也可进行 (可选)
- **[步骤名]** — [描述] → [技能名]

### 之后要做
- [下一个必需步骤] ([技能名])
```

格式规则：✓ 确认完成，→ 当前必需下一步，~ 可选步骤。

---

## 步骤8: 门禁警告

如果当前阶段所有必需步骤都已完成（或接近完成），提示用户准备运行 gate-check。

---

## 步骤9: 升级路径

如果用户表现出困惑，添加：

```
需要更多细节？
- project-stage-detect — 完整差距分析
- gate-check — 正式就绪检查
- start — 从零重新定位
```

---

## 协作协议

- **绝不自动运行下一个技能**。推荐它，让用户自行调用。
- **对于MANUAL步骤要询问**而非假设完成。
- **匹配用户语气** — 如果他们焦虑，只给一个行动而非六个。
- **一个主要推荐** — 用户离开时应清楚知道一件要做的事。

Verdict: **COMPLETE** — 已确定下一步。
