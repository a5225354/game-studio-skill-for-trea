# Architecture Decision — 架构决策记录(ADR)技能

触发条件:
- 用户说"架构决策" / "ADR" / "技术选型" / "architecture decision"
- 用户提供了决策标题

---

当用户触发此技能时:

## 1. 确定 ADR 编号

通过扫描 `docs/architecture/` 目录中已有的 ADR 文件确定下一个编号。

## 2. 收集上下文

阅读相关代码和已有 ADR。

## 3. 引导用户完成决策

如果仅凭标题不足以明确决策内容，使用 AskUserQuestion 提出澄清问题。

## 4. 生成 ADR

```markdown
# ADR-[NNNN]: [标题]

## 状态
[拟议 | 已采纳 | 已弃用 | 被 ADR-XXXX 取代]

## 日期
[决策日期]

## 背景
### 问题描述
[我们要解决什么问题？]

### 约束条件
- [技术约束]
- [时间约束]
- [资源约束]

## 决策
[具体的技术决策内容]

### 关键接口
[此决策创建的 API 契约或接口定义]

## 备选方案
### 备选方案 1：[名称]
- **描述**：[工作原理]
- **优点**：[优势]
- **缺点**：[劣势]
- **否决原因**：[为何未被采用]

### 备选方案 2：[名称]
- **描述**：[工作原理]
- **优点**：[优势]
- **缺点**：[劣势]
- **否决原因**：[为何未被采用]

## 影响
### 正面影响
- [此决策带来的良好结果]

### 负面影响
- [接受的权衡和代价]

### 风险
- [可能出现的问题及应对措施]

## 迁移计划
[如何从当前状态过渡到目标状态]

## 验证标准
[如何判断此决策是否正确]

## 相关决策
- [关联 ADR 链接]
```

## 5. 保存

保存至 `docs/architecture/adr-[NNNN]-[slug].md`。

## 协作协议

1. 读取上下文和已有 ADR
2. 引导用户澄清决策内容
3. 展示完整 ADR 草稿
4. 写入前获得用户批准
