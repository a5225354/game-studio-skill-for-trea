# Tech Debt — 技术债务管理技能

触发条件:
- 用户说"技术债务" / "tech debt" / "TODO" / "FIXME" / "债务扫描"
- 子命令: scan / add / prioritize / report

---

当用户触发此技能时:

## 1. 解析子命令

- `scan` — 扫描代码库中的技术债务指标
- `add` — 手动添加新的技术债务条目
- `prioritize` — 重新排列现有债务登记册的优先级
- `report` — 生成当前债务状态的汇总报告

---

## 2. scan — 扫描技术债务

搜索代码库中的债务指标:
- TODO 注释（统计并分类）
- FIXME 注释
- HACK 注释
- @deprecated 标记
- 重复代码块
- 超过 500 行的文件
- 超过 50 行的函数

对每个发现进行分类:
- **架构债务**：错误的抽象、缺失的模式、耦合问题
- **代码质量债务**：重复、复杂度、命名、缺失类型
- **测试债务**：缺失的测试、不稳定的测试、未覆盖的边界情况
- **文档债务**：缺失的文档、过时的文档、未文档化的 API
- **依赖债务**：过时的包、废弃的 API、版本冲突
- **性能债务**：已知的慢路径、未优化的查询、内存问题

更新 `docs/tech-debt-register.md`。

---

## 3. add — 添加债务条目

使用 AskUserQuestion 收集：描述、类别、受影响的文件、预估修复工作量、不修复的影响。追加到登记册。

---

## 4. prioritize — 优先级排序

读取债务登记册，按以下公式评分：`(不修复的影响 * 遇到频率) / 修复工作量`。按优先级重新排序。

---

## 5. report — 生成报告

生成汇总统计：
- 按类别统计的总条目数
- 总预估修复工作量
- 趋势方向（增长 / 稳定 / 减少）
- 标记超过 3 个冲刺未处理的条目

---

## 债务登记册格式

```markdown
## 技术债务登记册
Last updated: [日期]
Total items: [N]

| ID | Category | Description | Files | Effort | Impact | Priority | Added |
|----|----------|-------------|-------|--------|--------|----------|-------|
| TD-001 | [类别] | [描述] | [文件] | [S/M/L/XL] | [Low/Med/High/Critical] | [分数] | [日期] |
```

## 规则

- 技术债务本身不是坏事 — 登记册追踪的是有意识的决定
- 每条债务条目必须说明接受它的原因
- 每个冲刺至少应运行一次 scan
- 超过 3 个冲刺未处理的条目应予以修复或有意识地接受

## 协作协议

1. 扫描或读取登记册
2. 分类和优先级排序
3. 展示报告/结果
4. 写入前获得用户批准
