# Perf Profile — 性能分析技能

触发条件:
- 用户说"性能分析" / "perf" / "优化" / "性能" / "perf profile"
- 用户指定了系统名称或 "full"

---

当用户触发此技能时:

## 1. 确定范围

- 如果是系统名称：专注于该特定系统
- 如果是 `full`：对所有系统进行全面分析

## 2. 读取性能预算

检查现有性能目标：
- 目标 FPS（例如 60fps = 16.67ms 帧预算）
- 内存预算（总量和按系统）
- 加载时间目标
- Draw Call 预算
- 网络带宽限制（如果是多人游戏）

## 3. 分析代码库

### CPU 分析目标
- `_process()` / `Update()` / `Tick()` 函数 — 列出所有并估算开销
- 热路径中的分配模式
- 每帧代码中的昂贵操作
- 未优化的游戏实体搜索/排序

### 内存分析目标
- 大型数据结构及其增长模式
- 纹理/资源内存占用估算
- 对象池 vs 实例化/销毁模式
- 泄漏的引用

### 渲染目标
- Draw Call 估算
- 过度绘制检查
- 着色器复杂度
- 缺失 LOD 或遮挡剔除

## 4. 生成报告

```markdown
## 性能分析：[系统或全面]
生成日期：[日期]

### 性能预算
| 指标 | 预算 | 估算当前值 | 状态 |
|------|------|-----------|------|
| 帧时间 | [16.67ms] | [估算值] | [正常/警告/超标] |
| 内存 | [目标] | [估算值] | [正常/警告/超标] |

### 发现的热点
| # | 位置 | 问题 | 预估影响 | 修复工作量 |
|---|------|------|---------|-----------|

### 优化建议（按优先级排序）
1. **[标题]** — [优化描述]
   - 位置：[文件:行号]
   - 预期收益：[估算值]
   - 方案：[实现方式]

### 快速见效（每个不到 1 小时）
- [简单优化 1]

### 需要进一步调查
- [需要实际运行时分析来确定的领域]
```

## 规则

- 不要先测量就优化 — 对性能的直觉是不可靠的
- 建议必须包含预估影响 — "让它更快"不具有可操作性
- 区分 CPU 瓶颈、GPU 瓶颈和 I/O 瓶颈
- 考虑最坏情况（最大实体数、最低配置硬件）

## 协作协议

1. 读取性能预算目标
2. 分析代码中的性能热点
3. 生成优先级排序的建议
4. 此技能为分析模式 — 展示发现，不修改代码
