# Playtest Report — 试玩报告技能

触发条件:
- 用户说"试玩报告" / "playtest" / "测试报告" / "试玩"

---

当用户使用 `new` 模式时，生成以下模板：

```markdown
# 试玩报告

## 会话信息
- **日期**：[日期]
- **构建版本**：[版本/提交]
- **时长**：[游玩时间]
- **测试者**：[姓名/ID]
- **平台**：[PC/主机/移动端]
- **输入方式**：[键鼠 / 手柄 / 触屏]
- **会话类型**：[首次游玩 / 回访 / 针对性测试]

## 测试重点
[本次测试的具体功能或流程]

## 第一印象（前 5 分钟）
- **理解目标了吗？** [是/否/部分]
- **理解操作了吗？** [是/否/部分]
- **情绪反应**：[投入/困惑/无聊/沮丧/兴奋]

## 玩法流程
### 做得好的地方
- [观察 1]

### 痛点
- [问题 1 — 严重程度：高/中/低]

### 困惑点
- [玩家在哪里感到困惑以及原因]

### 令人惊喜的时刻
- [什么让玩家感到惊喜或满意]

## 遇到的 Bug
| # | 描述 | 严重程度 | 可复现？ |
|---|------|---------|---------|

## 功能专项反馈
### [功能 1]
- **理解用途了吗？** [是/否]
- **觉得有趣吗？** [是/否]
- **建议**：[测试者的建议]

## 定量数据（如有）
- **死亡次数**：[次数和位置]
- **各区域用时**：[细分]

## 总体评价
- **愿意再玩吗？** [是/否/可能]
- **难度**：[太简单 / 刚好 / 太难]
- **节奏**：[太慢 / 不错 / 太快]

## 本次会话的前 3 优先事项
1. [最重要的发现]
2. [第二优先]
3. [第三优先]
```

当使用 `analyze` 模式时，读取原始笔记，与现有设计文档交叉引用，并用结构化的发现填充模板。标记任何与设计意图冲突的试玩观察。

## 协作协议

1. 生成模板或分析原始笔记
2. 展示填充后的报告
3. 写入前获得用户批准
