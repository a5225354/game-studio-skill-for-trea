# Launch Checklist — 全面上线就绪验证技能

触发条件:
- 用户说"上线清单" / "launch checklist" / "上线前" / "上线就绪" / "最终检查"
- 用户提供了发布日期或 "dry-run"

---

当用户触发此技能时:

> **仅限显式调用**：此技能仅在用户明确表达上线验证意图时执行。

## 1. 读取参数

获取发布日期或 `dry-run` 模式。演练模式仅生成清单，不创建签字条目。

## 2. 收集项目上下文

- 读取技术栈、目标平台和团队结构
- 读取最新的里程碑（`production/milestones/`）
- 读取已有的发布清单（`production/releases/` 如有）

## 3. 扫描代码库健康度

- 统计 TODO、FIXME、HACK 注释及其位置
- 检查生产代码中是否遗留了调试输出
- 检查占位资产（搜索 `placeholder`、`temp_`、`WIP_`）
- 检查硬编码的测试/开发值（localhost、测试凭据、调试标志）

## 4. 生成发布清单

```markdown
# 发布清单: [游戏标题]
目标发布: [日期或 DRY RUN]
生成时间: [日期]

---

## 1. 代码就绪

### 构建健康
- [ ] 所有目标平台干净构建通过
- [ ] 零编译器警告
- [ ] 所有单元测试通过
- [ ] 所有集成测试通过
- [ ] 性能基准测试达标
- [ ] 无内存泄漏
- [ ] 构建体积在平台限制内
- [ ] 构建版本号正确设置并在源代码管理中打标签

### 代码质量
- [ ] TODO 数量: [N]（发布要求为零，或有记录的例外）
- [ ] FIXME 数量: [N]（发布要求为零）
- [ ] HACK 数量: [N]（每项必须有书面说明）
- [ ] 生产代码中无调试输出
- [ ] 无硬编码的开发/测试值
- [ ] 所有功能开关已设置为生产值
- [ ] 所有关键路径的错误处理已覆盖
- [ ] 崩溃报告已集成并验证

### 安全
- [ ] 源代码中无暴露的 API 密钥或凭据
- [ ] 存档数据已加密
- [ ] 网络通信已加密（TLS/DTLS）
- [ ] 反作弊措施已启用（如有多人模式）
- [ ] 所有服务器端点已做输入验证
- [ ] 隐私政策合规已验证

---

## 2. 内容就绪

### 资产
- [ ] 所有占位美术资产已替换为最终资产
- [ ] 所有占位音频已替换为最终音频
- [ ] 音频混音已完成并经音频总监审批
- [ ] 所有 VFX 已打磨并通过性能验证
- [ ] 无缺失或损坏的资产引用
- [ ] 资产命名规范已执行

### 文本与本地化
- [ ] 所有面向玩家的文本已完成校对
- [ ] 无硬编码字符串（全部已外部化以支持本地化）
- [ ] 所有支持语言的翻译已完成并验证
- [ ] 文本在所有语言中适配 UI
- [ ] 制作人员名单完整、准确且为最新

### 游戏内容
- [ ] 所有关卡/地图可从头到尾通关
- [ ] 教程流程已完成并通过新玩家测试
- [ ] 所有成就/奖杯已实现并测试
- [ ] 存档/读档在所有游戏状态下正常工作
- [ ] 难度设置已平衡并测试
- [ ] 游戏结尾/制作人员名单序列已完成

---

## 3. 质量保证

### 测试
- [ ] 完整回归测试套件已通过
- [ ] 零 S1 级缺陷未关闭
- [ ] 零 S2 级缺陷未关闭（或有记录的例外）
- [ ] 浸泡测试通过（连续运行 8 小时以上）
- [ ] 所有关键用户路径在每个平台上已测试
- [ ] 边界情况已测试（存储满、无网络、挂起/恢复）

### 平台认证
- [ ] PC: Steam/Epic/GOG SDK 要求已满足
- [ ] 主机: TRC/TCR/Lotcheck 提交材料已准备
- [ ] 移动端: App Store/Play Store 规范已合规
- [ ] 无障碍: 最低标准已满足
- [ ] 年龄评级已获取

### 性能
- [ ] 最低配置硬件上达到目标帧率
- [ ] 所有平台加载时间在预算内
- [ ] 所有平台内存使用在预算内
- [ ] 关键游戏时刻无帧率卡顿

---

## 4. 商店与分发

### 商店页面
- [ ] 商店页面文案已定稿并校对
- [ ] 截图为最新版本
- [ ] 预告片为最新版本并已审批
- [ ] 系统配置要求准确（PC）
- [ ] 所有地区的定价已配置

### 法务
- [ ] 最终用户许可协议（EULA）已定稿并经法务审批
- [ ] 隐私政策已发布并正确链接
- [ ] 第三方许可归属说明已完成
- [ ] 音乐/音频授权已验证
- [ ] GDPR/CCPA 合规已验证

---

## 5. 基础设施

### 服务器（如有多人/在线功能）
- [ ] 生产服务器已配置并通过负载测试
- [ ] 自动扩缩容已配置并测试
- [ ] 数据库备份已配置
- [ ] DDoS 防护已启用
- [ ] 监控和告警已配置

### 数据分析与监控
- [ ] 数据分析管道已验证并正常接收数据
- [ ] 崩溃报告已启用且仪表盘可访问
- [ ] 关键指标已追踪：DAU、会话时长、留存率、崩溃率
- [ ] 关键阈值告警已配置

---

## 6. 社区与市场

### 社区就绪
- [ ] 社区准则已发布
- [ ] Discord/论坛/社交媒体频道已建立
- [ ] 常见问题（FAQ）和已知问题页面已准备
- [ ] 客服邮箱/工单系统已启用

### 市场推广
- [ ] 发布预告片已发布
- [ ] 媒体/创作者评测密钥已分发
- [ ] 社交媒体发布帖已排期
- [ ] 发布版本的补丁说明已发布

---

## 7. 运营

### 团队就绪
- [ ] 发布后 72 小时内的值班表已排定
- [ ] 事件响应手册已由团队审阅
- [ ] 回滚计划已记录并测试
- [ ] 紧急修复管道已测试
- [ ] 发布问题沟通计划已制定

---

## 上线 / 不上线决策

**整体状态**: [就绪 / 未就绪 / 有条件通过]

### 阻塞项
[列出发布前必须解决的所有事项]

### 有条件项
[列出具已记录的变通方案或已接受风险的事项]

### 签字要求
- [ ] 创意总监 — 内容与体验质量
- [ ] 技术总监 — 技术健康度与稳定性
- [ ] QA 负责人 — 质量与测试覆盖率
- [ ] 制片人 — 进度与整体就绪度
- [ ] 发布经理 — 构建与部署就绪度
```

## 5. 保存清单

保存到 `production/releases/launch-checklist-[date].md`。

## 6. 输出摘要

总项目数、阻塞项数量、有条件项数量、有未完成章节的部门，以及文件路径。

## 协作协议

1. 全面扫描代码库健康度
2. 生成 7 大部门完整验证清单
3. 使用 AskUserQuestion 确认关键未自动验证项
4. 展示完整清单草稿
5. 写入前获得用户批准
