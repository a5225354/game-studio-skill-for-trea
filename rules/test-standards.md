# 测试标准规则 (Test Standards Rules)

- **测试命名**: `test_[系统]_[场景]_[预期结果]`（如 `test_damage_calculation_with_armor_reduces_output`）
- **结构**: Arrange / Act / Assert — 准备、执行、验证
- **无外部状态依赖**: 每个测试独立运行，不依赖其他测试的执行顺序
- **自行清理**: 测试后恢复所有修改的状态
- **一行对应一文件**: 每个测试文件对应一个源文件
- **覆盖要求**:
  - 所有公共 API 必须有单元测试
  - 所有边界情况必须有测试
  - 公式必须有测试（包括极端值）
- **性能测试**: 关键路径必须有性能测试（如 10000 次计算 < 1ms）
