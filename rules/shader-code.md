# 着色器代码规则 (Shader Code Rules)

- **命名约定**: `[类型]_[系统]_[描述]`（如 `post_color_grading`、`vfx_fire_particle`）
- **性能目标**: 
  - 顶点着色器：低/中复杂度设备上 < 100 条指令
  - 片段着色器：移动端 < 20 条纹理采样
- **跨平台**: 使用引擎的着色器抽象层，避免平台特定语法
- **变体控制**: 限制着色器变体数量 — 在编译时使用预处理器宏而非运行时分支
- **精度**: 移动端使用适当的精度限定符（lowp、mediump、highp）
- **后处理**: 后处理效果必须可单独开关，并测量性能影响
