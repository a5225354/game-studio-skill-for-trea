# 原型代码规则 (Prototype Code Rules)

**标准有意放宽** — 原型用于探索和验证，而非生产代码。

- 可以随意硬编码值 — 不需要从配置加载
- 使用占位资源 — 不需要最终美术/音频
- 跳过错误处理 — 聚焦于核心机制
- 使用最简单可行的方案 — 不优化，先验证
- 复制代码而非从正式代码导入 — 严格隔离

**必需项**:
- 每个原型文件必须以注释标记: `// PROTOTYPE - NOT FOR PRODUCTION`
- 必须包含 README 文件，记录:
  - 核心假设
  - 成功标准
  - 发现和结论
  - 建议（推进/转向/放弃）
- **绝不要**从正式源文件导入
- **绝不要**在正式代码中导入原型代码

原型总工作量应控制在 1-3 天内。如果范围膨胀，重新评估。
