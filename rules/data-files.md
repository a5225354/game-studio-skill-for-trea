# 数据文件规则 (Data Files Rules)

- **JSON 有效性**: 所有 `assets/data/` 中的文件必须是有效的 JSON 格式
- **命名约定**: `[system]_[subtype].json`（如 `combat_damage.json`、`economy_prices.json`）
- **Schema 规则**: 每个数据文件类型必须有预期的结构：
  - 数组元素的字段必须一致
  - 数值字段必须有文档化的范围
  - 引用的 ID 必须指向存在的实体
- **不可修改性**: 数据文件在运行时不应被修改（只读加载）
- **版本控制**: 数据文件必须包含版本字段
- **注释**: 使用 `"_comment"` 字段为关键值提供上下文
- **本地化分离**: 本地化文本不应放在数据文件中 — 使用专门的本地化文件
